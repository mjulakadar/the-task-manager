Angular Components

1. Introduction to Components

In Angular, a component is a building block for the UI. It consists of:

  • HTML template: Defines the view.
  • CSS styles: Defines the look and feel.
  • TypeScript class: Defines the behavior and state.

Components are used to encapsulate functionality and are defined using the @Component decorator.

2. Life Cycle of Components

Angular components have a well-defined life cycle. Understanding these life cycle hooks allows you to tap into key events and perform specific tasks at different stages of a component's existence.


Key Life Cycle Hooks

1. Modules: Angular applications are modular and organized into NgModules. Each module can contain components, services, and other modules. The AppModule is the root module that bootstraps the application.

2. Components: Components are the building blocks of Angular applications. They consist of an HTML template, CSS styles, and a TypeScript class that defines the component's behavior.

3. Templates: Templates define the view for a component. They use HTML along with Angular's template syntax to render data and bind to component properties.

4. Services and Dependency Injection: Services are used to share data and functionality across components. Angular uses dependency injection to provide services to components.

5. Routing: Angular's Router module allows for navigation between different views or components within the application.

6. Directives: Directives are special markers in the template that tell Angular to do something with the DOM elements. There are three types: structural directives, attribute directives, and component directives.

7. Pipes: Pipes are used to transform data in templates. For example, you can use a pipe to format dates or numbers.
