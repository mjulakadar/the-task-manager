Angular Routing and Navigation

1. The Basics of Routing

Angular's Router module allows you to navigate between different views or components in your application. It provides a way to define routes and handle navigation through a powerful and flexible system.


Setting Up Routing

1. Import RouterModule and Routes: First, import RouterModule and Routes from @angular/router in your module.

2. Add RouterOutlet in AppComponent: In your root component (AppComponent), add a `router-outlet` tag where you want the routed components to be displayed.

2. Router Outlet

The router-outlet directive acts as a placeholder for the routed components. When the route changes, the Angular router dynamically updates the content within the router-outlet.

3. Route Parameters [Parameter Value: {{userId}}]

Route parameters allow you to pass data through the URL. They are useful for dynamic navigation where you need to display different data based on the route parameter.


Defining Route Parameters

1. Define Route with Parameters: Use the :parameter syntax in the route path.

2. Access Route Parameters: Use the ActivatedRoute service to access route parameters in your component.

3. Child Routes

Child routes allow you to create nested routes, which is useful for creating more complex routing structures.


Defining Child Routes

1. Define Child Routes: Use the children property in the route configuration.

2. Access Child Routes: Child routes will be rendered within the `router-outlet` of the parent component.

3. Router Guards

Router guards are used to control access to routes. They can prevent navigation to or from a route based on specific conditions.


Types of Router Guards

1. CanActivate: Determines if a route can be activated.

2. CanActivateChild: Determines if child routes can be activated.

3. CanDeactivate: Determines if a route can be deactivated.

4. Resolve: Pre-fetch data before activating a route.


Using Router Guards

1. Add Guard to Route: Apply the guard to routes in the route configuration.

2. Implement Guard Logic: In your guard, implement the logic to determine if navigation should be allowed.
