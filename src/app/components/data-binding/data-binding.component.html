Data Binding

1. One Way Data Binding

One-way data binding in Angular involves binding a component's data to the view. This ensures that any changes in the component's data automatically update the view, but changes in the view do not affect the component's data. This is typically achieved using interpolation or property binding.

{{message}}
2. Property Binding

Property binding is used to bind an attribute of an HTML element to a property in the component. It is done using the `[ ]` syntax.

3. String Interpolation

Interpolation is used to bind a component's property to the view. It is done using the `{{InterpololationBracket}}` syntax.

{{interpolationMessage}}
4. Two Way Data Binding

Two-way data binding in Angular allows for data to be synchronized between the component and the view. Changes in the component's data are reflected in the view, and changes in the view are reflected in the component. This is typically achieved using the ngModel directive.

To use ngModel, you need to import the FormsModule in your Angular application. You can your property using [(ngModel)]="propertyName" syntax.

5. Event Binding

Event binding in Angular allows you to listen for and respond to user actions, such as clicks, key presses, mouse movements, and other DOM events. This is achieved using the (event) syntax, where event is the type of DOM event you want to bind to.
